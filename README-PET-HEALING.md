## 🎯 玩法流程

### 1. 宠物受伤
```
对战失败 → 宠物HP低于50%
        → 自动标记 is_injured = True
        → 宠物页面显示红色警告卡片
```

### 2. 开始治疗
```
点击"💊 立即治疗"按钮
→ 进入治疗学习页面
→ 显示:
  - 当前HP进度条
  - 宠物图片(灰度 + 虚弱动画)
  - 还需答对几题
  - 单词选择题(4选1)
```

### 3. 答题治疗
```
答对1题 → +5 HP ✨
         → HP条绿色填充动画
         → 宠物图片逐渐变亮
         → 鼓励文案变化

答错 → 显示正确答案
     → 不恢复HP
     → 继续下一题
```

### 4. 恢复健康
```
HP达到80%以上
→ 弹出"✨ 满血复活"动画
→ 3秒后自动返回宠物页面
→ 可以继续对战了！
```

## 📊 数据流

### 数据库字段
```sql
-- user_pets 表新增字段
current_hp INTEGER DEFAULT 120,      -- 当前HP
is_injured BOOLEAN DEFAULT FALSE     -- 是否受伤
```

### API接口
```
GET  /api/v1/student/pet/healing-status    获取治疗状态
GET  /api/v1/student/pet/healing-words      获取治疗单词
POST /api/v1/student/pet/heal               提交答题(恢复HP)
```

### 关键逻辑
```python
# 对战结束时标记受伤
if pet.current_hp < max_hp * 0.5:
    pet.is_injured = True

# 答题治疗
if is_correct:
    pet.current_hp = min(max_hp, pet.current_hp + 5)
    
    # HP恢复到80%以上,解除受伤
    if pet.current_hp >= max_hp * 0.8:
        pet.is_injured = False
```

## 💡 设计思考

### 为什么选择"每题+5 HP"?

**平衡性考虑**:
- 太少(+2 HP): 需要答30题,太枯燥
- 太多(+10 HP): 需要答6题,太简单
- **+5 HP**: 需要约10-15题,正好一个学习单元

### 为什么恢复到80%而不是100%?

**用户体验考虑**:
- 100%需要答更多题(20+题)
- 80%是"健康线",可以对战了
- 剩余20%鼓励再学习一点点

### 为什么有灰度渐变动画?

**情感设计**:
- 灰暗 = 虚弱,引发同情心
- 渐变 = 治疗见效,正向反馈
- 彩色 = 恢复健康,成就感
