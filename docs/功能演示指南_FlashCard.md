# FlashCard学习模式功能演示指南

## 如何体验新功能

### 前提条件

1. 确保后端服务器运行在 http://localhost:8000
2. 确保前端服务器运行在 http://localhost:5173
3. 使用学生账号登录: `student` / `123456`

---

## 新功能演示步骤

### 1. 开始学习

1. 登录后进入学生仪表板
2. 选择一个单词本
3. 选择一个单元
4. 点击"FlashCard学习"按钮

**预期效果**:
- 系统自动创建学习会话
- 如果有薄弱单词,会在顶部显示提醒

---

### 2. 查看薄弱单词提醒

**触发条件**:
- 该单元中存在掌握度 < 3 的单词

**显示内容**:
```
⚠️ 发现薄弱单词!
当前单元有 X 个单词掌握度较低,需要加强练习哦!
```

**特点**:
- 橙红色渐变背景
- 5秒后自动消失
- 可点击关闭按钮手动关闭

---

### 3. 查看单词掌握度

**显示位置**: 卡片上方,难度星级旁边

**显示内容**:
- 蓝紫色渐变背景
- 6个圆点表示0-5级掌握度
- 黄色实心圆点 = 已达到的级别
- 白色半透明圆点 = 未达到的级别
- 数字显示当前级别 (如 "3/5")

**何时显示**:
- 首次学习的单词: 不显示掌握度
- 有学习历史的单词: 显示上次的掌握度

---

### 4. 学习单词

1. 查看单词正面 (英文单词 + 音标)
2. 点击音频按钮听发音
3. 点击卡片翻转查看释义
4. 选择"认识"或"不认识"

**后台记录**:
- ✅ 答题时间 (从显示单词到选择的时间)
- ✅ 答题结果 (认识/不认识)
- ✅ 学习模式 (flashcard)
- ✅ 单词ID

---

### 5. 完成学习

完成所有单词后:

1. 系统批量提交所有学习记录
2. 后端自动计算每个单词的掌握度
3. 后端自动计算下次复习时间
4. 更新学习会话统计
5. 显示完成对话框

**数据更新**:
- 每个单词的 `mastery_level` 会根据本次答题更新
- `next_review_at` 会根据掌握度设置复习时间
  - Level 4-5: 7天后复习
  - Level 2-3: 3天后复习
  - Level 0-1: 1天后复习

---

## 如何验证数据已保存

### 方法1: 再次学习同一单元

1. 返回单元列表
2. 再次点击同一单元的"FlashCard学习"
3. 观察单词卡片上方的掌握度显示

**预期结果**:
- 上次选择"认识"的单词掌握度会提升
- 上次选择"不认识"的单词掌握度较低

### 方法2: 使用API文档查询

1. 访问 http://localhost:8000/docs
2. 找到 `GET /api/v1/student/mastery` 端点
3. 点击"Try it out"
4. 添加Authorization: `Bearer YOUR_TOKEN`
5. 执行查询

**返回示例**:
```json
[
  {
    "id": 1,
    "user_id": 1,
    "word_id": 101,
    "total_encounters": 2,
    "correct_count": 1,
    "wrong_count": 1,
    "mastery_level": 1,
    "flashcard_correct": 1,
    "flashcard_wrong": 1,
    "last_practiced_at": "2025-11-21T13:00:00",
    "next_review_at": "2025-11-22T13:00:00"
  }
]
```

### 方法3: 查询薄弱单词

1. 访问 http://localhost:8000/docs
2. 找到 `GET /api/v1/student/weak-words` 端点
3. 添加 `unit_id` 参数
4. 执行查询

**预期结果**:
- 返回掌握度 < 3 的所有单词
- 包括详细的学习统计数据

## 掌握度等级说明

| 等级 | 条件 | 含义 | 复习间隔 |
|------|------|------|----------|
| Level 5 | 正确率 ≥95% 且遇到 ≥5次 | 完全掌握 | 7天 |
| Level 4 | 正确率 ≥85% 且遇到 ≥4次 | 基本掌握 | 7天 |
| Level 3 | 正确率 ≥75% 且遇到 ≥3次 | 一般掌握 | 3天 |
| Level 2 | 正确率 ≥60% 且遇到 ≥2次 | 初步掌握 | 3天 |
| Level 1 | 遇到 ≥1次 | 刚接触 | 1天 |
| Level 0 | 未学习 | 未接触 | - |

## 测试场景建议

### 场景1: 首次学习新单元

**目标**: 验证学习记录创建

**步骤**:
1. 选择一个从未学过的单元
2. 完成所有单词的学习
3. 查询掌握度API

**预期**:
- 所有单词都有掌握度记录
- 选择"认识"的单词掌握度 = 1
- 选择"不认识"的单词掌握度 = 0

### 场景2: 重复学习提升掌握度

**目标**: 验证掌握度算法

**步骤**:
1. 学习一个单元,所有单词都选"认识"
2. 立即再次学习该单元,再次都选"认识"
3. 观察掌握度变化

**预期**:
- 第一次学习后: mastery_level = 1
- 第二次学习后: mastery_level = 2 (正确率100%, 遇到2次)

### 场景3: 薄弱单词提醒

**目标**: 验证薄弱单词提醒功能

**步骤**:
1. 学习一个单元,故意选择几个"不认识"
2. 退出后重新进入该单元
3. 观察是否显示薄弱单词提醒

**预期**:
- 显示薄弱单词数量
- 5秒后自动消失

## 开发者调试技巧

### 1. 查看学习记录提交日志

打开浏览器控制台 (F12),在完成学习时会看到:
```
学习记录提交成功!
```

如果提交失败:
```
提交学习记录失败: [错误信息]
```

### 2. 查看网络请求

在浏览器开发者工具的 Network 标签中:

- 学习开始时: `POST /api/v1/student/sessions`
- 学习结束时: `POST /api/v1/student/records`
- 学习结束时: `PUT /api/v1/student/sessions/{id}`
- 切换单词时: `GET /api/v1/student/mastery/{word_id}`

### 3. 查看状态更新

在 FlashCardLearning.tsx 中添加 console.log:
```typescript
console.log('Current mastery:', currentWordMastery);
console.log('Word answers:', wordAnswers);
console.log('Correct/Wrong:', correctCount, wrongCount);
```

## 已知限制

1. **离线学习**:
   - 学习记录需要在线提交
   - 如果网络断开,记录会丢失
   - 建议: 未来可以添加本地缓存

2. **实时更新**:
   - 掌握度在学习结束时批量更新
   - 单词翻转后不会立即看到掌握度变化
   - 需要重新开始学习才能看到新的掌握度

3. **其他学习模式**:
   - Quiz、Spelling、FillBlank 模式尚未集成
   - 这些模式暂时不记录学习数据
