# 英语学习助手 - 完整项目说明

## 📋 项目概览

这是一个专为中小学生设计的英语单词学习系统,采用前后端分离架构,集成AI能力,提供趣味化的学习体验。

| 功能模块 | 学生端 | 老师端 | AI集成 | 状态 |
|---------|--------|--------|--------|------|
| 单词管理 | ✅ 查看 | ✅ 录入/编辑 | ✅ 生成例句 | 已完成 |
| 卡片记忆 | 📝 待开发 | - | - | 设计完成 |
| 选择题测试 | 📝 待开发 | - | ✅ 生成干扰项 | 设计完成 |

| 例句填空 | 📝 待开发 | - | ✅ 生成例句 | 设计完成 |
| 智能出题 | 📝 待开发 | ✅ 生成试卷 | ✅ 薄弱点分析 | API已完成 |
| 学习进度 | 📝 待开发 | ✅ 查看报告 | - | 数据库已设计 |
| 成就系统 | 📝 待开发 | - | - | 数据库已设计 |

## 🏗️ 技术架构

### 后端技术栈
```
语言: Python 3.9+
框架: FastAPI (异步高性能)
数据库: SQLite (轻量级,易部署)
ORM: SQLAlchemy 2.0 (异步支持)
AI: OpenAI/Claude API
验证: Pydantic
```

### 前端技术栈 (推荐)
```
框架: React 18 + TypeScript
样式: Tailwind CSS
动画: Framer Motion
状态: Zustand
请求: React Query
音效: Howler.js
```

### 部署方案
```
开发环境: uvicorn
生产环境: Gunicorn + Nginx
容器化: Docker + Docker Compose
云服务: 阿里云/腾讯云
```

## 🗄️ 数据库设计亮点

### 1. 灵活的单词结构
- 支持一词多义 (`word_definitions` 表)
- 支持多标签分类 (`word_tags` 表)
- 支持音频/图片资源

### 2. 智能学习追踪
- `user_word_progress` 表记录每个单词的掌握度
- 支持间隔重复算法 (`next_review_time`)
- 详细的学习日志 (`learning_records`)

### 3. AI内容缓存
- `ai_cache` 表避免重复调用AI
- 大幅降低API成本
- 提高响应速度

### 4. 成就系统
- 游戏化激励机制
- 学习日历打卡
- 可扩展的成就定义

## 🤖 AI集成详解

### AI服务架构
```python
AIService (基类)
├── generate_example_sentence()      # 生成例句
├── generate_distractors()           # 生成干扰项
├── explain_mistake()                # 解释错误
├── recommend_words()                # 推荐单词
├── generate_exam_questions()        # 生成试卷
└── analyze_weak_points()            # 分析薄弱点
```

### 支持的AI模型
- **OpenAI GPT-4**: 质量最高,成本较高
- **Claude 3 Sonnet**: 性价比高,适合批量任务
- **通义千问/文心一言**: 国内部署,待集成

### 成本优化策略
1. **三级缓存**
   - 内存缓存 (即时)
   - 数据库缓存 (持久)
   - 静态资源预生成 (高频内容)

2. **智能降级**
   - 简单任务用小模型
   - 复杂任务用大模型
   - 缓存未命中时才调用API

3. **批量处理**
   - 合并多个请求
   - 减少API调用次数

```dockerfile
# Dockerfile示例
FROM python:3.9-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["gunicorn", "app.main:app", "-w", "4", "-k", "uvicorn.workers.UvicornWorker"]
```

```yaml
# docker-compose.yml
version: '3.8'
services:
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=sqlite:///data/english_helper.db
    volumes:
      - ./data:/app/data

  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend
```

## 💡 最佳实践

### 单词录入
1. 每个单词至少1个释义
2. 主要释义标记为 `is_primary=true`
3. 例句简单易懂,15词以内
4. 使用标签分类,方便检索

### AI使用
1. 开发阶段可不配置AI,不影响基础功能
2. 生产环境建议配置至少一个AI服务
3. 定期清理过期缓存
4. 监控API用量和成本

### 前端开发
1. 移动优先设计
2. 60fps动画流畅度
3. 离线功能支持
4. 无障碍访问 (a11y)

## 🐛 常见问题排查

### 问题: 数据库初始化失败
```bash
# 检查schema文件路径
ls -la database_schema.sql

# 手动初始化
sqlite3 backend/english_helper.db < database_schema.sql
```

### 问题: AI功能报错
```python
# 检查配置
python -c "from app.core.config import settings; print(settings.OPENAI_API_KEY)"

# 测试网络
curl https://api.openai.com/v1/models
```

### 问题: 导入依赖失败
