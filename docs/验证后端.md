# 🧪 后端验证步骤

## 步骤1: 启动后端服务

### Mac/Linux:
```bash
cd backend
chmod +x start.sh
./start.sh
```

### Windows:
```bash
cd backend
start.bat
```

## 步骤2: 检查服务是否正常

打开浏览器访问:
- **健康检查**: http://localhost:8000/health
- **API文档**: http://localhost:8000/docs

应该看到:
```json
{"status": "healthy"}
```

## 步骤3: 导入示例数据

**新开一个终端窗口**,运行:

```bash
cd backend
source venv/bin/activate  # Mac/Linux
# 或
venv\Scripts\activate     # Windows

python seed_data.py
```

预期输出:
```

✅ happy - 导入成功

```

## 步骤5: 手动测试API (可选)

### 5.1 访问API文档
打开 http://localhost:8000/docs

你会看到Swagger UI,可以直接测试所有API

### 5.2 测试创建单词
在API文档中找到 `POST /api/v1/words/`,点击 "Try it out"

填入示例数据:
```json
{
  "word": "computer",
  "phonetic": "/kəmˈpjuːtə/",
  "difficulty": 3,
  "grade_level": "初中",
  "definitions": [
    {
      "part_of_speech": "n.",
      "meaning": "计算机;电脑",
      "example_sentence": "I use a computer every day.",
      "example_translation": "我每天使用电脑。",
      "is_primary": true
    }
  ],
  "tags": ["科技", "日常用品"]
}
```

点击 "Execute",应该看到 201 响应。

### 5.3 测试获取单词
找到 `GET /api/v1/words/`,点击 "Try it out"

参数:
- limit: 10
- grade_level: 小学

点击 "Execute",应该看到单词列表。

### 5.4 测试AI功能 (需要配置API Key)

**如果你还没配置AI:**
1. 编辑 `backend/.env` 文件
2. 填入 `OPENAI_API_KEY=sk-xxx` 或 `ANTHROPIC_API_KEY=sk-ant-xxx`
3. 重启后端服务

**测试生成例句:**
找到 `POST /api/v1/ai/generate-example`,填入:
```json
{
  "word": "happy",
  "meaning": "快乐的",
  "difficulty": "primary-school"
}
```

应该看到AI生成的例句和翻译。

## 常见问题

### ❌ 端口8000被占用
```bash
# 查找占用端口的进程
lsof -i :8000  # Mac/Linux
netstat -ano | findstr :8000  # Windows

# 杀死进程或换端口
uvicorn app.main:app --reload --port 8001
```

### ❌ 模块导入错误
```bash
# 确保在虚拟环境中
which python  # 应该显示 venv/bin/python

# 重新安装依赖
pip install -r requirements.txt
```

### ❌ 数据库初始化失败
```bash
# 删除旧数据库
rm english_helper.db

# 重启服务,会自动创建新数据库
```

### ❌ AI功能报错
- 检查 `.env` 文件中的API Key是否正确
- 检查网络能否访问OpenAI/Claude
- AI功能是可选的,不影响其他功能
