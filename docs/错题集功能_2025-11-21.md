# 错题集功能开发总结 2025-11-21

## 功能概述

错题集功能允许学生查看、复习和针对性练习所有答错的单词,帮助学生系统性地攻克薄弱点,提升学习效率。

---

## 核心特性

### 1. 智能错题识别
- 自动从学习记录中识别所有答错的单词
- 支持多种学习模式的错题统计(FlashCard、Quiz、Spelling、FillBlank)
- 区分"未解决"和"已掌握"的错题

### 2. 多维度统计
- **总错题数**: 所有答错过的单词(去重)
- **未解决错题**: 掌握度 < 4 的错题
- **已掌握错题**: 掌握度 ≥ 4 的错题
- **今日练习**: 今天练习过的错题数量
- **本周练习**: 最近7天练习过的错题数量
- **按模式统计**: FlashCard、Quiz、Spelling、FillBlank 各模式的错题数

### 3. 智能排序算法
错题列表按以下优先级排序:
1. **最近错误次数**(降序) - 最近7天错误次数多的优先
2. **掌握度**(升序) - 掌握度低的优先
3. **总错误次数**(降序) - 历史错误次数多的优先

### 4. 快速开始练习
- 一键开始针对性练习
- 自动准备最需要复习的错题
- 支持选择不同的学习模式
- 限制练习数量(默认20个)

## 技术实现

### 后端API

#### 1. 数据模型 (`backend/app/schemas/mistake_book.py`)

```python
class MistakeWordDetail:
    """错题单词详情"""
    - word_id, word, phonetic, meaning, part_of_speech
    - total_mistakes          # 总错误次数
    - recent_mistakes         # 最近7天错误次数
    - last_mistake_at         # 最后一次答错时间
    - mastery_level           # 当前掌握度 0-5
    - correct_count, wrong_count
    - flashcard_wrong, quiz_wrong, spelling_wrong, fillblank_wrong
    - is_resolved             # 是否已解决 (掌握度 >= 4)

class MistakeBookStats:
    """错题集统计"""
    - total_mistakes          # 总错题数
    - unresolved_mistakes     # 未解决的错题数
    - resolved_mistakes       # 已解决的错题数
    - flashcard_mistakes, quiz_mistakes, spelling_mistakes, fillblank_mistakes
    - today_practice_count, week_practice_count
```

#### 2. API端点 (`backend/app/api/v1/student/mistake_book.py`)

**获取错题集统计**:
```
GET /api/v1/student/mistake-book/stats
```
返回错题集的整体统计信息。

**获取错题列表**:
```
GET /api/v1/student/mistake-book/words?only_unresolved=true&unit_id=1
```
参数:
- `only_unresolved`: 只显示未解决的错题(默认true)
- `unit_id`: 筛选指定单元的错题(可选)

**开始错题练习**:
```
POST /api/v1/student/mistake-book/practice
Body: {
  "learning_mode": "flashcard",
  "limit": 20,
  "only_unresolved": true,
  "unit_id": null
}
```
返回智能排序后的练习单词列表。

**标记错题为已解决**:
```
DELETE /api/v1/student/mistake-book/words/{word_id}
```
手动将某个错题标记为已掌握(提升掌握度到4级)。

## 数据流程

### 错题生成流程:

```
1. 学生在学习时答错单词
   ↓
2. 学习记录API记录 is_correct = false
   ↓
3. 错题集API从 learning_records 表筛选错题
   ↓
4. 关联 word_mastery 表获取掌握度
   ↓
5. 计算统计数据并返回
```

### 错题练习流程:

```
1. 学生访问错题集页面
   ↓
2. 查看错题统计和列表
   ↓
3. 选择学习模式(FlashCard/Quiz/Spelling/FillBlank)
   ↓
4. 点击"开始练习"
   ↓
5. 后端智能排序错题(最需要复习的优先)
   ↓
6. 返回练习单词列表
   ↓
7. 跳转到对应的学习模式
   ↓
8. 学生练习,系统记录新的学习记录
   ↓
9. 掌握度自动更新
   ↓
10. 错题可能变为"已掌握"状态
```

## 智能算法详解

### 1. 错题判定
```sql
SELECT DISTINCT word_id
FROM learning_records
WHERE user_id = ? AND is_correct = false
```

### 2. 未解决错题判定
```sql
错题 AND (mastery_level IS NULL OR mastery_level < 4)
```

### 3. 已解决错题判定
```sql
错题 AND mastery_level >= 4
```

### 4. 智能排序算法
```python
sorted(
    mistake_words,
    key=lambda w: (
        -w.recent_mistakes,  # 最近错误次数多的优先(降序)
        w.mastery_level,     # 掌握度低的优先(升序)
        -w.total_mistakes    # 总错误次数多的优先(降序)
    )
)
```

**排序逻辑说明**:
- 优先级1: 最近7天错过很多次的单词 → 说明当前掌握最差
- 优先级2: 掌握度低的单词 → 长期掌握不好
- 优先级3: 历史上错过很多次的单词 → 一直是难点

## 数据库查询优化

### 1. 使用DISTINCT去重
```sql
SELECT DISTINCT(LearningRecord.word_id)
```
确保每个单词只计算一次。

### 2. JOIN优化
```sql
.join(WordDefinition, Word.id == WordDefinition.word_id)
.outerjoin(WordMastery, ...)
```
一次查询获取所有需要的数据。

### 3. 分组聚合
```sql
.group_by(Word.id, WordDefinition.id, WordMastery.mastery_level, ...)
```
在数据库层面完成统计。
