# 学习薄弱分析规则说明

## 一、薄弱单词的定义

### 1. 判定标准
系统使用**掌握度(mastery_level)**作为核心指标来判定薄弱单词:

- **薄弱单词**: `mastery_level < 3`
- **掌握单词**: `mastery_level >= 4`
- **中等掌握**: `3 <= mastery_level < 4`

### 2. 掌握度的计算
掌握度是一个0-5的整数,基于以下因素综合计算:

```python
掌握度 = f(正确次数, 错误次数, 总遇见次数, 时间间隔)
```

**具体规则**:
- 初始掌握度: `0`
- 答对一次: `+1` (最高不超过5)
- 答错一次: `-1` (最低不低于0)
- 长时间未复习: 自然衰减

## 二、薄弱点分析维度

### 1. 错误率 (Error Rate)
```
错误率 = (wrong_count / total_attempts) × 100%
准确率 = 100% - 错误率
```

**排序优先级**:
1. 掌握度最低的单词 (mastery_level ASC)
2. 最后练习时间最早的单词 (last_practiced_at ASC)

### 2. 学习模式分析
系统追踪单词在不同学习模式下的表现:

| 学习模式 | 字段名 | 说明 |
|---------|--------|------|
| 🎴 卡片模式 | flashcard_wrong | 翻卡学习时的错误次数 |
| 📝 选择题 | quiz_wrong | 多选题练习的错误次数 |
| ✍️ 拼写练习 | spelling_wrong | 拼写测试的错误次数 |
| 📋 填空题 | fillblank_wrong | 填空练习的错误次数 |

**返回数据示例**:
```json
{
  "learning_modes": ["quiz", "spelling"],
  "error_count": 8,
  "total_attempts": 12,
  "accuracy_rate": 33.3
}
```

### 3. 时间因素
- `last_error_at`: 最后一次答错的时间
- `last_practiced_at`: 最后一次练习的时间
- 长时间未复习的单词会被优先推荐复习

## 三、AI试卷生成策略

### 1. 难度评估
系统根据学生整体准确率自动推荐难度:

| 准确率范围 | 推荐难度 | 说明 |
|-----------|---------|------|
| < 60% | easy (简单) | 基础巩固 |
| 60% - 80% | medium (中等) | 常规练习 |
| > 80% | hard (困难) | 挑战提升 |

### 2. 题型分布策略
根据学生在不同学习模式下的错误情况,智能分配题型:

**默认分布** (总20题):
- 选择题 (choice): 8题 (40%)
- 填空题 (fill_blank): 6题 (30%)
- 拼写题 (spelling): 4题 (20%)
- 阅读理解 (reading): 2题 (10%)

**动态调整规则**:
```python
if spelling_error_rate > 50%:
    增加拼写题比例到 30%

if quiz_error_rate > 60%:
    增加选择题比例到 50%

if fillblank_error_rate > 50%:
    增加填空题比例到 35%
```

### 3. 单词选择
- 优先选择**掌握度最低**的15个单词
- 如果薄弱单词不足,补充**中等掌握**的单词
- 确保选词覆盖学生的主要薄弱点

## 四、数据统计字段说明

### 学生学习统计 (StudentLearningStats)

| 字段 | 类型 | 说明 |
|------|------|------|
| `words_learned` | int | 已学单词总数 |
| `total_learning_time` | int | 总学习时长(秒) |
| `study_sessions` | int | 学习会话数/学习天数 |
| `weak_words_count` | int | 薄弱单词数(掌握度<3) |
| `accuracy_rate` | float | 整体准确率(%) |
| `last_active` | datetime | 最后活动时间 |

### 薄弱单词详情 (StudentWeakPoint)

| 字段 | 类型 | 说明 |
|------|------|------|
| `word` | str | 单词 |
| `meaning` | str | 中文释义 |
| `mastery_level` | int | 掌握度(0-5) |
| `error_count` | int | 错误次数 |
| `total_attempts` | int | 总尝试次数 |
| `accuracy_rate` | float | 该单词准确率(%) |
| `learning_modes` | list | 出错的学习模式 |
| `last_error_at` | datetime | 最后错误时间 |

## 五、教学建议生成规则

系统根据分析结果自动生成个性化教学建议:

### 1. 针对错误率高的学习模式
```
if 'spelling' in error_modes:
    建议: "加强拼写训练,可以使用听写练习"

if 'quiz' in error_modes:
    建议: "多做选择题练习,加深对词义的理解"
```

### 2. 针对掌握度分布
```
if weak_words_count > 20:
    建议: "薄弱单词较多,建议每天集中攻克5-10个"

if average_mastery < 2.0:
    建议: "整体掌握度较低,建议从基础单词开始复习"
```

### 3. 针对学习频率
```
if days_since_last_study > 7:
    建议: "已经一周未学习,建议尽快开始复习以防遗忘"

if study_sessions < 5:
    建议: "学习次数较少,建议增加练习频率"
```

## 六、优化建议

### 当前系统的改进空间:

1. **更精细的掌握度算法**
   - 考虑时间衰减因素
   - 引入间隔重复算法 (SM2/SM18)
   - 根据单词难度调整评分标准

2. **更智能的题型分配**
   - 基于机器学习预测最佳练习模式
   - 个性化题型偏好设置
   - 动态难度调整

3. **学习路径推荐**
   - 基于薄弱点生成学习计划
   - 推荐相似单词对比学习
   - 提供记忆技巧和助记符

4. **数据可视化**
   - 学习进度曲线图
   - 掌握度热力图
   - 错误模式分析图表
