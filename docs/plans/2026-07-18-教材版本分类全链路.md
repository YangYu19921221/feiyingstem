# 教材版本分类(series)全链路实施计划

## 目标

单词本增加"教材版本"分类(人教版/苏教版/…/机构自定义),打通:
建表迁移 → 上传/创建 → 编辑 → 教师列表筛选 → 学生书架筛选 → 作业/分配选择器筛选 → 存量回填 → 部署。

已确认决策:①只做"版本"单维度(学段从 grade_level 推导);②org_admin 可自定义分类(挂 org_id 隔离,同名拒绝,每机构上限 20)。

## A. 数据层

1. **models/word.py**
   - `WordBook` 加 `series = Column(String(30), nullable=True)`
   - 新模型 `BookSeries`:id / name(String(30)) / org_id(nullable, NULL=平台预置) / sort_order / created_at
2. **core/database.py init_db 迁移**(照抄 org_id 迁移先例,幂等):
   - `ALTER TABLE word_books ADD COLUMN series VARCHAR(30)`(捕获"duplicate column"跳过)
   - `CREATE TABLE IF NOT EXISTS book_series(...)`
   - 预置 7 项(人教版/苏教版/外研版/牛津译林版/北师大版/课外读物/校本教材),仅当表空时插入
   - **存量回填**(幂等,只动 series IS NULL 的行):名称含"人教"→人教版、"苏教"→苏教版、"外研"→外研版、"牛津/译林"→牛津译林版、"北师大"→北师大版、"飞鹰"→校本教材;其余留空由老师手动补
3. **core/tenancy.py**:`TENANT_MODELS` 追加 `(BookSeries, True)`(可空锚点,NULL=平台共享,与 WordBook 同款读写安全网)

## B. 后端 API

4. **schemas/word.py**:`WordBookBase`、`WordBookUpdate` 加 `series: Optional[str]`(Response 继承自动带上)
5. **words.py**
   - `POST /words/book-series`:建自定义分类。守卫:`role in ('admin','org_admin')`(admin 建平台级 org_id=NULL,org_admin 建机构级);同名查重(平台+本机构范围);机构上限 20
   - `GET /words/book-series`:列表(tenancy 自动过滤出 平台预置+本机构),按 sort_order,id 排序;所有教师可读
   - `create_word_book` / 更新端点写入 `series`
6. **teacher/units.py** 工作簿导入:请求 schema 加 `series`,建书时写入
7. **student/progress.py**:`StudentBookListItem` 加 `series` 并填充
8. **admin/content.py**:books 列表响应带 series(查当前序列化方式,多数是 dict 直拼,补一个 key)

## C. 前端

9. **api 层**:`teacher.ts` 的 `TeacherWordBook` 加 series;新增 `getBookSeries()` / `createBookSeries(name)`;学生书架类型加 series
10. **TeacherBooks.tsx**(单词本管理):
    - 创建/编辑表单加"教材版本"下拉(选项=GET book-series);org_admin/admin 额外显示"➕新增分类"内联输入
    - 列表顶部加版本筛选 chips(全部|人教版|…|未分类),与现有搜索、学段分组叠加
    - 卡片上显示 series 徽章
11. **TeacherUnitManagement.tsx**(Excel 上传导入):导入表单加"教材版本"下拉
12. **ScopeSelector.tsx**(作业/分配共用):搜索框旁加版本下拉,与搜索叠加过滤;`BookOption` 加 `series?`;两个调用方(TeacherHomework/TeacherBookAssignment)传入的书列表带 series
13. **StudentDashboard_New.tsx** 书架:搜索框旁加版本 chips 过滤(数据来自 StudentBookListItem.series)

## D. 测试(本地全链路)

14. 重启本地后端 → 验证迁移:表结构、预置 7 项、存量回填命中数(本地 26 本书预计 ≥20 本自动归类)
15. Playwright 教师端:建自定义分类"XX冲刺班" → 建书选它 → 列表筛选 → 编辑改分类;Excel 导入表单带版本
16. Playwright:作业创建 ScopeSelector 版本+搜索叠加过滤;学生书架 chips 过滤
17. 越权测试:普通 teacher POST book-series 应 403

## E. 部署(风险控制)

18. **逐文件 md5 对比生产**:models/word.py、core/database.py、core/tenancy.py、schemas/word.py、words.py、teacher/units.py、student/progress.py、admin/content.py——**任一有漂移即停,改用远程 anchor patch**(classes.py 有三方漂移前科)
19. 备份 → scp → 生产 py_compile → 重启(init_db 自动跑迁移) → 验证:`GET /words/book-series` 返回预置项、抽查 `word_books.series` 回填、服务健康
20. 前端 dist 备份 → rsync → 线上 bundle 验证
21. 回滚预案:新列可空、新表独立,回滚只需还原 .py 文件重启,数据无损

## 工作量与顺序

后端(A+B)→ 本地迁移验证 → 前端(C)→ 全链路测试(D)→ 部署(E)。
