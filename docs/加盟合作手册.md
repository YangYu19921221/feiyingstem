## 一、我们是什么

我们不是又一个"背单词 APP",而是一套 **"教-学-测-评-赛"完整闭环的智能英语教学系统**。

中小机构不用自己开发、不用请技术团队,开箱就能拥有大机构级别的数字化教学能力和游戏化留生工具:

- 老师端:班级管理、内容自建、精细化分配、作业布置、实时课堂监控、投屏大屏
- 学生端:闪卡 / 听写 / 拼写 / 选择 / 填空 / 例句 / 句子背诵 / 单元考试 等多种学习模式
- 游戏化:宠物养成、真人实时 PK 对战、全自动晋级赛、段位 / 成就 / 排行榜
- AI 能力:自动出题、智能组卷、千人千卷、错因讲解、学习周报
- 发音:剑桥真人发音 + 双引擎语音评测,不请外教也能练口语
- 家长端:绑定孩子看学习看板 + 周报
- 招生工具:AI 英语测评漏斗,自带获客能力

一句话:**别人加盟给你一套课件,我们给你一套能自己跑起来的数字化教学系统。**

## 二、核心卖点(对机构老板讲)

机构老板最关心三件事:**招得来、留得住、教得省**。系统正好对着这三点。

### ① 帮你招生(获客)
- **AI 英语水平测评漏斗**:家长 / 学生做一次匿名测评 → 生成专业深度报告 → 自动进入你的招生线索池,老师跟进转化。
- 地推、朋友圈、进校、家长会都能用,是一个自带的免费获客入口。

### ② 帮你留生(续费)
- **宠物养成 + 真人实时 PK + 全自动晋级赛 + 段位 / 成就 / 排行榜**——把背单词做成孩子天天想上线的"游戏",黏性远超普通打卡类 APP。
- **防划水 / 防作弊 + 切屏专注监控 + 家长端周报**:家长能看到孩子真实在学、学了多少,续费时你有据可讲。

### ③ 帮你省人省课时(降本)
- **AI 自动出题、智能组卷、按错题千人千卷、AI 错因讲解、AI 周报**——备课、批改、写反馈的活儿系统全包。
- **真人级发音评测**(剑桥真人发音 + 双引擎语音打分),不请外教也能练口语跟读。
- 老师端班级管理、精细化分配、实时课堂监控,一个老师能带的学生数直接翻倍。

## 五、AI 使用说明与额度

系统的 AI 分两类管理,机构无需关心技术细节,只需了解额度即可。

### A 类:一次生成、全校复用(不计入机构用量)

这些内容同一个单词在全校范围内只生成一次,之后所有学生共享,**几乎不产生额外成本,不设上限**:

- AI 例句生成、选择题干扰项
- 单词音标 / 释义 / 词条补全
- 单词本封面插图

### B 类:按人头实时消耗(设合理上限)

以下功能按每个学生 / 每位老师实时调用,为保障系统长期稳定,设置合理上限(正常教学完全够用,仅防滥用):

| 功能 | 触发场景 | 额度上限 |
|---|---|---|
| AI 错因讲解 | 学生答错时讲解为什么错 | 每生每日 20 次 |
| AI 智能组卷 / 个性化试卷 | 老师一键出卷 | 每位老师每日 10 份 |
| AI 竞赛批量出题 | 老师批量生成竞赛题 | 每位老师每月 200 题 |
| AI 学习周报 | 家长 / 老师查看 | 每生每周 1 份(自动缓存,重复查看不计) |
| AI 招生测评报告 | 潜在学员做测评 | 每加盟点每月 500 份 |

> 超出上限:当日 / 当月额度用完后功能暂停至下一周期,或按需单独购买加量包(对外可作为增值项)。

### 关于宠物对战

宠物对战、PK 竞技场的"AI 对手"为智能算法生成(动态匹配学员水平),**不消耗大模型额度**,可无限畅玩,不占用量。

> **内部说明(不对外)**:上述上限为建议值。落地前需在代码中实际加上限流逻辑——当前系统对 AI 调用尚无任何配额控制,任何用户可无限触发。上线加盟前务必先补齐限流(重点:错因讲解、个性化组卷、竞赛出题三项),否则"额度"只是纸面承诺。

# 附录:对内成本 / 毛利测算(不对外)

> 以下仅供内部定价参考,不放进给加盟商的版本。

## 单个学生一年的真实 AI 成本

成本几乎完全取决于**后台配置的模型**(存在数据库 `ai_providers` 表,需登录管理后台查看)。按系统实际调用频次估算:

| 后台配置模型 | 单生年 AI 成本 | 100 赠送账号年硬成本 | 备注 |
|---|---|---|---|
| 通义千问 / 国产模型 | 几元以内 | < ¥1,000 | 代码中 DashScope 配置暗示线上很可能用此类 |
| Claude Sonnet | 约 ¥30 | 约 ¥3,000 | |
| GPT-4-turbo | 约 ¥60-70 | 约 ¥6,000-7,000 | 个性化组卷 16000 token,成本最高 |

**关键动作**:登录管理后台确认 `ai_providers` 里默认启用的模型。若为国产模型,送 100 账号成本可忽略,标准档稳赚。

## 标准档毛利粗算(假设线上用国产模型)

| 项目 | 首年 | 次年起 |
|---|---|---|
| 收入(招 100 人) | 39,800 | 6,800 |
| AI 硬成本 | < 1,000 | < 1,000 |
| 服务器 / 带宽(分摊) | 视规模 | 视规模 |
| 培训 / 物料(一次性) | 人力成本 | — |
| **毛利** | **≈ 3.5 万+** | **≈ 5,000+** |

## 三个必做事项(定价才踏实)

1. **确认线上模型**:后台查 `ai_providers` 默认模型 → 决定送账号是否亏。
2. **补齐 AI 限流**:当前代码对 AI **零配额控制**。上线加盟前必须给三个高频/高价功能加上限:
   - `explain_mistake`(错因讲解):学生端高频、零缓存 → 加每生每日上限 + 持久缓存
   - `generate_personalized_exam`(个性化组卷):单次 16000 token → 加每老师每日上限
   - `generate_competition_question`(竞赛出题):循环调用 → 加每月总量上限
3. **修缓存**:当前缓存是进程内存缓存(重启即失效、多进程不共享),数据库 `ai_cache` 表未被使用 → 建议改 Redis 或落库,让 A 类内容真正"一次生成全校复用",否则实际成本高于上表估算。
