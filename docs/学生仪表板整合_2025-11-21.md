### 2. 配色方案

| 模块 | 配色 | 用途 |
|-----|------|------|
| 欢迎横幅 | from-primary to-secondary | 主题色渐变 |
| 学习数据 | 蓝/绿/橙/紫 各自50色 | 区分不同数据类型 |

| 快捷功能 | 白色卡片 + 悬停渐变 | 简洁统一 |
| 成就预览 | 黄-橙渐变(已解锁) / 灰色(未解锁) | 激励感 |

### 3. 动画效果

| 动画类型 | 应用位置 | 效果 |
|---------|---------|------|
| 淡入上移 | 欢迎横幅 | `y: -20 → 0` |
| 淡入下移 | 各个模块 | `y: 20 → 0` |
| 缩放渐进 | 数据卡片、快捷功能 | `scale: 0.8 → 1` |
| 旋转弹出 | 成就徽章 | `scale: 0 → 1` |
| 悬停缩放 | 可点击元素 | `scale: 1 → 1.05` |
| 按压缩小 | 按钮 | `scale: 1 → 0.95` |

### 4. 交互反馈

- **悬停**: 阴影加深 (`shadow-sm → shadow-md → shadow-lg`)
- **渐变背景**: 悬停时显示彩色渐变 (`opacity: 0 → 0.1`)
- **鼠标指针**: 可点击元素显示 `cursor-pointer`
- **颜色变化**: 链接文字 `text-primary → text-secondary`

```tsx
const [stats, setStats] = useState<LearningOverview | null>(null);
const [achievements, setAchievements] = useState<UserAchievements | null>(null);
```

## 🐛 已知问题

### 1. 数据为静态
**问题**: 学习数据概览和成就预览使用硬编码数据
**影响**: 无法反映真实学习情况
**优先级**: 🔴 高
**解决方案**: 集成 `getLearningOverview()` 和 `getMyAchievements()` API

### 2. 打卡天数硬编码
**问题**: 欢迎横幅显示 "第7天打卡" 为固定值
**影响**: 所有学生看到相同提示
**优先级**: 🟡 中
**解决方案**: 从 `stats.current_streak` 读取

### 3. 排名百分比硬编码
**问题**: "超越了85%的学习者" 为固定文案
**影响**: 缺乏个性化,可能不准确
**优先级**: 🟢 低
**解决方案**: 后端计算排名,或移除此文案
