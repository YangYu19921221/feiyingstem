## 🚨 WebSocket连接失败解决方案

你遇到的 `WebSocket connection failed` 错误是因为缺少依赖。按以下步骤解决:

### 方法1: 使用虚拟环境(推荐)

```bash
cd /Users/apple/Desktop/英语助手/backend

# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate

# 安装所有依赖(包括websockets和redis)
pip install -r requirements.txt

# 启动后端
uvicorn app.main:app --reload
```

### 方法2: 直接安装(使用--break-system-packages)

```bash
cd /Users/apple/Desktop/英语助手/backend

# 安装WebSocket依赖
python3 -m pip install websockets redis --break-system-packages

# 启动后端

```

## 📚 竞赛模式题目来源说明

### 重要说明

**竞赛模式不需要单独录入题目!**

竞赛系统会**自动使用所有已录入的单词**作为题库:

1. ✅ 教师通过"单词录入"功能录入的所有单词
2. ✅ 教师通过"批量导入"导入的所有单词
3. ✅ 已分配到单元的所有单词

### 题目生成机制

```
单词库中的单词
    ↓
自动生成选择题
    ↓
学生答题时随机抽取
    ↓
自动生成3个干扰选项
    ↓
形成完整的选择题
```

**示例**:

如果单词库中有单词 `happy`:
- **单词**: happy
- **音标**: /ˈhæpi/
- **释义**: 快乐的
- **难度**: 3

**自动生成题目**:
```
题目: 请选择'happy'的正确释义

A. 快乐的   ✅ (正确答案)
B. 悲伤的   (干扰项,自动从其他形容词中选择)
C. 生气的   (干扰项)
D. 害怕的   (干扰项)
```

## 📖 使用流程

### 教师端操作流程

1. **录入单词** (必需)
   ```
   访问: /teacher/words
   录入单词及释义
   ```

2. **创建单词本和单元** (可选)
   ```
   访问: /teacher/books
   创建单词本 → 创建单元 → 添加单词到单元
   ```

3. **查看竞赛题库** (可选)
   ```
   访问: /teacher/competition
   查看题库统计 → 预览题目效果
   ```

4. **学生即可开始竞赛!**
   ```
   学生访问: /student/competition
   系统自动从题库抽取题目
   ```

### 学生端操作流程

1. **登录学生账号**
   ```
   访问: /login
   ```

2. **进入竞赛模式**
   ```
   方法1: 在仪表盘点击"🏆 竞赛模式"卡片
   方法2: 直接访问 /student/competition
   ```

3. **开始答题**
   ```
   系统自动加载题目
   → 选择答案
   → 查看得分反馈
   → 排名实时更新
   → 继续下一题
   ```

## 💡 常见问题

### Q1: WebSocket连接失败怎么办?

**A**: 安装websockets依赖:
```bash
source venv/bin/activate
pip install websockets redis
```

### Q2: 竞赛模式没有题目怎么办?

**A**:
1. 确保教师已经录入了单词
2. 确保单词有释义
3. 访问 `/teacher/competition` 查看题库统计

### Q3: 题目的干扰选项是怎么生成的?

**A**: 系统自动从其他同词性的单词中随机选择3个释义作为干扰项

### Q4: 可以控制哪些单词出现在竞赛中吗?

**A**: 当前所有录入的单词都可用于竞赛。未来可以添加"启用/禁用"功能

### Q5: 学生能看到所有题目吗?

**A**: 不能,系统每次随机抽取题目,保证公平性
